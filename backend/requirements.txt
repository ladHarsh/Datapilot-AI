#  Security Requirements (merged from security_requirements.txt)
# ──────────────────────────────────────────────────────────────────────────────
#
# 1. AUTHENTICATION & SESSION
# - Every request (except public routes) MUST include a valid JWT.
# - JWT must be bound to an active server-side session.
# - Logged-out tokens MUST be blacklisted until expiration.
# - Brute-force protection must block accounts after 5 failed attempts.
#
# 2. SQL VALIDATION & EXECUTION
# - All queries must be strictly SELECT-only (read-only).
# - No DDL (DROP, ALTER, CREATE) or destructive DML (DELETE, UPDATE, INSERT) allowed.
# - Queries must be checked for injection patterns using advanced regex.
# - Complexity scoring must block JOIN-heavy or resource-intensive queries.
# - Schema validation must ensure tables exist before execution.
#
# 3. PROTECTION LAYER
# - Global execution timeout of 30 seconds for all DB operations.
# - Mandatory LIMIT clause on every analytical query (max 10,000 rows).
# - Access to internal 'users', 'passwords', and 'audit' tables restricted to Admins.
# - All API responses must include secure HTTP headers (CSP, HSTS, etc.).
#
# 4. AUDIT & MONITORING
# - Every query execution must be logged with UserID, SQL, and Status.
# - Suspicious activity (repeated blocks) must trigger alerts.
# - Performance metrics (execution time) must be tracked.
#
# 5. EXPORTS
# - Data export (CSV, Excel, PDF) restricted to 'Admin' and 'Analyst' roles.
# - Export size limited to 50MB per file.
# - All exported filenames must be sanitized to prevent path traversal.
