## Instructions
- Analyse the supplied SQL query and identify any **inefficient patterns** (e.g., missing indexes, unnecessary sub‑queries, redundant joins, N+1 style loops, scanning large tables, lack of filters).
- For each issue, **suggest a specific improvement**:
  - Propose an index creation statement when filtering or joining on non‑indexed columns.
  - Recommend query restructuring (e.g., replace sub‑queries with joins, use CTEs for readability, add explicit `JOIN` types).
  - Highlight potential N+1 problems and how to batch them.
  - Suggest using `WITH` (CTE) if it clarifies complex logic.
- Output a **numbered list** of suggestions. Keep each suggestion concise (one sentence).
- If the query is already well‑optimised, simply respond with the exact phrase:
  `Query looks well‑optimized`
- Do **not** return any additional text, markdown, or code fences.
